# MULTI-MARKET — как добавить рынок

> Актуализировано 2026-07-02. Принцип: **один бренд, один код, одна база — много рынков.**
> Новый рынок = ДАННЫЕ, не новый код. Сейчас активен только `phuket`.

Рынок определяется таблицей **`markets`** и полем **`market_id`** в tours/knowledge/packages
(старая модель «по полю city» упразднена).

## Чек-лист нового рынка (например, Паттайя из архива)
1. **БД**: `UPDATE markets SET active = true WHERE id = 'pattaya';` + туры/знания с `market_id='pattaya'` (уже лежат в БД).
2. **Сайт**: лендинг из archive/landings + запись в shared/markets.js (`active: true`).
3. **Бот**: блок рынка в KOTE_SOUL (n8n/live/main-bot-ai-agent.json → «✍️ Собрать промпт») — тексты сохранены в git-истории.
4. **PWA**: город уже вшит в app.html (splash-выбор).
5. Проверка: get_kote_context фильтрует по рынку — спросить бота про туры нового города.

Возврат рынков — этап Q4'26–Q1'27, см. ROADMAP.md.
