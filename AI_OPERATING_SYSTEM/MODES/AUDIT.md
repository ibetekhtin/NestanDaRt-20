## ЧТО ДЕЛАТЬ

### 1. Сбор информации
- [ ] Прочитать все ключевые файлы (CLAUDE.md, MASTER_PROMPT.md, README.md)
- [ ] Прочитать docker-compose.yml
- [ ] Прочитать все requirements.txt / package.json
- [ ] Прочитать конфигурации nginx
- [ ] Прочитать .env.example (без секретов)
- [ ] Прочитать все Python-файлы в app/backend/
- [ ] Прочитать все JS/JSX файлы в hq/
- [ ] Прочитать все HTML/CSS/JS в nestandart-phuket/
- [ ] Прочитать platform/app.html

### 2. Построение карты
- [ ] Создать граф зависимостей (какой файл импортирует какой)
- [ ] Определить точки входа (main.py, index.html, App.jsx)
- [ ] Определить листья (файлы без зависимостей)
- [ ] Найти все маршруты (nginx, React Router, FastAPI)
- [ ] Найти все API endpoints

### 3. Поиск проблем
- [ ] **Dead code:** функции/классы/компоненты, которые нигде не вызываются
- [ ] **Duplicate code:** одинаковые функции, компоненты, стили, конфиги
- [ ] **Unused imports:** импорты, которые не используются
- [ ] **Unused dependencies:** пакеты в requirements.txt / package.json, которые не импортируются
- [ ] **Unused endpoints:** API-руты, которые нигде не вызываются
- [ ] **Unused pages:** страницы, на которые нет ссылок
- [ ] **Temporary files:** файлы с именами temp, tmp, test, debug, backup
- [ ] **Prototype files:** прототипы, которые не в продакшене
- [ ] **Old configs:** устаревшие конфиги (старые версии nginx, docker и т.д.)
- [ ] **Hardcoded secrets:** секреты в коде (не в .env)
- [ ] **Security issues:** открытые порты, слабые политики, публичный Swagger

### 4. Документирование
- [ ] Создать `REPORTS/AUDIT_TEMPLATE.md` с результатами
- [ ] Для каждой проблемы указать:
  - Файл
  - Строка
  - Тип проблемы
  - Кто использует (если использует)
  - Риски удаления
  - Confidence score (0-100%)
