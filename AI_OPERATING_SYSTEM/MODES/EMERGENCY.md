## НАЗНАЧЕНИЕ

Быстрое восстановление после сбоя.

**Забыть про красоту.**  
**Думать только о починке.**  
**Минимальные изменения.**  
**Максимальная скорость.**

---

## ПРАВИЛА

### 1. Скорость > Идеальность
- [ ] Не рефакторить код
- [ ] Не улучшать архитектуру
- [ ] Не добавлять комментарии "для ясности"
- [ ] Только починить и уйти

### 2. Минимальность
- [ ] Изменить только то, что сломано
- [ ] Не трогать соседние модули
- [ ] Не оптимизировать "пока я тут"
- [ ] Не исправлять "заодно" другие баги

### 3. Безопасность
- [ ] Не ломать данные
- [ ] Не ломать конфигурацию
- [ ] Иметь план отката
- [ ] Фиксировать каждое изменение

---

## ПРОЦЕСС

### Шаг 1: Диагностика (5 минут)
- [ ] Что упало? (конкретный симптом)
- [ ] Где упало? (логи, ошибки)
- [ ] Когда упало? (после какого изменения)
- [ ] Кто受影响? (пользователи, клиенты, бот)

### Шаг 2: Быстрый фикс
- [ ] Найти минимальное изменение, которое починит
- [ ] Применить его
- [ ] Проверить: симптом исчез
- [ ] Проверить: ничего другое не сломалось

### Шаг 3: Верификация
- [ ] Основной функционал работает
- [ ] Нет новых ошибок в логах
- [ ] Клиенты могут пользоваться

### Шаг 4: Документация
- [ ] Записать что случилось
- [ ] Записать что починил
- [ ] Записать как предотвратить повторно

## ЧТО ЗАПРЕЩЕНО

| Запрещено | Почему |
|-----------|--------|
| Рефакторить код | Нет времени |
| Менять архитектуру | Нет времени |
| Добавлять новую функциональность | Нет времени |
| Исправлять "заодно" другие баги | Риск сломать больше |
| Удалять код "для порядка" | Риск сломать |
| Делать "правильно" вместо "быстро" | Клиенты ждут |

## ПЛАН ОТКАТА

В emergency режиме откат = возврат к предыдущему рабочему состоянию:

```bash
# Быстрый откат файлов
git checkout HEAD~1 -- <файлы>

# Быстрый откат Docker
docker compose rollback <сервис>

# Быстрый откат БД
psql -U postgres -d nestandart < backup_latest.sql

# Быстрый откат n8n
docker exec kote-n8n n8n import:workflow --backup=backup_latest.json
```

## КОНТРОЛЬНЫЕ ВОПРОСЫ

Перед каждым изменением:
1. **Это починит проблему?** (не "может помочь")
2. **Это минимальное изменение?** (не "правильное")
3. **Это не сломает данные?** (бэкап есть)
4. **Это можно откатить?** (git revert работает)
5. **Симптом исчез?** (проверено)

Если хотя бы один ответ «нет» — **НЕ ВЫПОЛНЯТЬ**.
