## ПРАВИЛА

### 1. Release — это не Build
- [ ] Не добавлять новые фичи
- [ ] Не исправлять баги (кроме блокирующих релиз)
- [ ] Только подготовка уже готового кода

### 2. Безопасность
- [ ] Перед деплоем — бэкап БД
- [ ] Перед деплоем — бэкап n8n workflows
- [ ] Иметь план отката
- [ ] Иметь команду отката (одна команда)

### 3. Верификация
- [ ] Все тесты проходят
- [ ] Линтер не ругается
- [ ] Сборка проходит без ошибок
- [ ] Docker образы собираются
- [ ] Миграции применены (если есть)

## ПРОЦЕСС

### Шаг 1: Подготовка
- [ ] `git status` — рабочий дерево чистое
- [ ] `git pull origin main` — актуальная main
- [ ] Создать ветку `release/vX.Y.Z` (если нужно)
- [ ] Обновить CHANGELOG.md
- [ ] Обновить версии (если есть)

### Шаг 2: Проверка
- [ ] Запустить тесты: `pytest` / `npm test`
- [ ] Запустить линтер: `flake8` / `eslint`
- [ ] Собрать фронтенд: `npm run build`
- [ ] Собрать Docker: `docker compose build`
- [ ] Проверить миграции: `alembic check` (если есть)
- [ ] Проверить .env на VPS (актуальный)

### Шаг 3: Бэкап
- [ ] Бэкап Supabase: `pg_dump` или через dashboard
- [ ] Бэкап n8n: `docker exec kote-n8n n8n export:workflow --all --backup`
- [ ] Бэкап файлов: `tar -czf backup.tar.gz /var/www/nestandart/`
- [ ] Сохранить метку времени: `date +%Y%m%d_%H%M%S`

### Шаг 4: Деплой
- [ ] Деплой бэкенда: `docker compose up -d --force-recreate`
- [ ] Деплой фронтенда: `scp` / `rsync` на VPS
- [ ] Деплой БАЗЫ: `npm run build && scp` (если нужно)
- [ ] Применить миграции (если есть)
- [ ] Перезагрузить nginx: `systemctl reload nginx`

### Шаг 5: Верификация
- [ ] Health check: `curl https://nestandart.online/health`
- [ ] Проверить markets: `curl https://nestandart.online/api/v1/markets`
- [ ] Проверить tours: `curl https://nestandart.online/api/v1/tours`
- [ ] Открыть сайт: https://nestandart.online
- [ ] Открыть PWA: https://app.nestandart.online
- [ ] Открыть HQ: https://baza.nestandart.online
- [ ] Проверить бот: отправить тестовое сообщение
- [ ] Проверить логи: `docker logs kote-backend --tail 50`

### Шаг 6: Мониторинг
- [ ] Ждать 5 минут
- [ ] Проверить error log: `tail -20 /var/log/nginx/error.log`
- [ ] Проверить backend logs: `docker logs kote-backend --tail 50`
- [ ] Проверить n8n: https://n8n.nestandart.online
- [ ] Проверить, что нет 500 ошибок

### Шаг 7: Откат (если нужно)
- [ ] Выполнить команду отката
- [ ] Проверить, что система работает
- [ ] Зафиксировать причину отката

## ПЛАН ОТКАТА

Каждый релиз должен иметь план отката:

```bash
# Откат бэкенда
git revert HEAD && docker compose build kote-backend && docker compose up -d  # отката-команды у compose нет kote-backend

# Откат фронтенда
git checkout HEAD~1 -- nestandart-phuket/
scp -r nestandart-phuket/ root@VPS:/var/www/nestandart/

# Откат БД
# БД в Supabase Cloud: восстановление — см. docs/RUNBOOK.md §Restore < backup_YYYYMMDD.sql

# Откат n8n
docker exec kote-n8n n8n import:workflow --backup=backup_YYYYMMDD.json
```

## ЧТО ЗАПРЕЩЕНО

| Запрещено | Почему |
|-----------|--------|
| Деплоить без бэкапа | Нельзя откатить |
| Деплоить без тестов | Могут быть регрессии |
| Деплоить без плана отката | Риск зависания |
| Менять код во время деплоя | Непредсказуемо |
| Деплоить в пятницу вечером | Не будет поддержки |
| Игнорировать ошибки в логах | Могут быть критические проблемы |

## КОНТРОЛЬНЫЕ ВОПРОСЫ

Перед деплоем:
1. **Есть бэкап?** (БД, n8n, файлы)
2. **Есть план отката?** (одна команда)
3. **Все тесты проходят?** (нет красных)
4. **Линтер чист?** (нет ошибок)
5. **Сборка проходит?** (нет ошибок)

Если хотя бы один ответ «нет» — **НЕ ДЕПЛОИТЬ**.
