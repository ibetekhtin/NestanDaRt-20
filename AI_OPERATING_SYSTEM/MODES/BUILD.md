# BUILD.md — РЕЖИМ ПОСТРОЕНИЯ
> Версия: 1.0

---

## НАЗНАЧЕНИЕ

Создание новой функциональности.

**Не удаляет существующий код.**  
**Не меняет архитектуру без необходимости.**  
**Только добавляет новое.**

---

## ПРАВИЛА

### 1. Входные данные
- [ ] Есть понятная задача (что построить)
- [ ] Есть требования (что должно делать)
- [ ] Есть примеры использования (если возможно)
- [ ] Нет конфликтов с существующим кодом

### 2. Безопасность
- [ ] Не ломать существующий API
- [ ] Не ломать существующий frontend
- [ ] Не ломать существующий backend
- [ ] Не ломать Docker
- [ ] Не ломать деплой
- [ ] Не ломать nginx
- [ ] Не ломать Supabase
- [ ] Не ломать n8n

### 3. Минимальность
- [ ] Создавать только необходимое
- [ ] Не создавать абстракции "на будущее"
- [ ] Не создавать сервисы "для масштабирования"
- [ ] Не создавать интерфейсы "на всякий случай"
- [ ] Использовать существующие паттерны

### 4. Интеграция
- [ ] Новый код интегрируется в существующую архитектуру
- [ ] Использует существующие зависимости
- [ ] Следует существующим конвенциям
- [ ] Не требует изменения других модулей

## ЧТО ЗАПРЕЩЕНО

| Запрещено | Почему |
|-----------|--------|
| Удалять старый код | Это BUILD, не REFACTOR |
| Менять архитектуру | Только добавление |
| Создавать абстракции "на будущее" | YAGNI |
| Переписывать существующие функции | Только новые |
| Добавлять зависимости без необходимости | Усложнение |
| Создавать новые папки "для порядка" | Без необходимости |

---

## ПРИНЦИПЫ

| Принцип | Описание |
|---------|----------|
| **YAGNI** | You Aren't Gonna Need It — не добавлять то, что может понадобиться |
| **KISS** | Keep It Simple, Stupid — проще лучше |
| **DRY** | Don't Repeat Yourself — не повторяться, но не переусложнять |
| **SOLID** | Только где это уменьшает сложность |

## КОНТРОЛЬНЫЕ ВОПРОСЫ

Перед каждым изменением:
1. **Это новая функциональность?** (не изменение старой)
2. **Это минимальная реализация?** (не "идеальная")
3. **Это не ломает существующее?** (все тесты проходят)
4. **Это использует существующие паттерны?** (не изобретает велосипед)
5. **Это можно откатить?** (`git revert` работает)

Если хотя бы один ответ «нет» — **НЕ ВЫПОЛНЯТЬ**.
