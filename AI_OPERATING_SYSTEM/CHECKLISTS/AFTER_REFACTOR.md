## НАЗНАЧЕНИЕ

Проверка после рефакторинга.

Убедиться, что ничего не сломалось.  
Зафиксировать результат.

---

## ПРОВЕРКИ

### 1. Запуск
- [ ] Backend запускается без ошибок
- [ ] Frontend собирается без ошибок
- [ ] Docker контейнеры поднимаются
- [ ] Нет ошибок в консоли браузера

### 2. Функциональность
- [ ] Основные страницы открываются
- [ ] API endpoints отвечают
- [ ] База данных доступна
- [ ] Бот отвечает (если затронуто)
- [ ] PWA работает (если затронуто)

### 3. Код
- [ ] Нет orphan imports (неиспользуемых импортов)
- [ ] Нет dead code (оставшегося мёртвого кода)
- [ ] Все функции вызываются
- [ ] Все классы используются

### 4. Производительность
- [ ] Размер бандла не вырос (если менял JS/CSS)
- [ ] Время загрузки не увеличилось
- [ ] Нет новых медленных запросов

### 5. Документация
- [ ] PROJECT_MAP.md обновлён (если менял архитектуру)
- [ ] DECISIONS.md обновлён (каждое удаление/объединение)
- [ ] CODE_AUDIT.md обновлён (если нужно)

---

## ЕСЛИ ЧТО-ТО СЛОМАЛОСЬ

1. **Немедленно откатить:** `git revert HEAD`
2. **Зафиксировать:** что сломалось, почему
3. **Проанализировать:** что упустил в аудите
4. **Исправить аудит:** добавить недостающие проверки
5. **Повторить:** только после исправления аудита
