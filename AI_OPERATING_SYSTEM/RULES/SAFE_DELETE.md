# SAFE_DELETE.md — ПРАВИЛА БЕЗОПАСНОГО УДАЛЕНИЯ
> Версия: 1.0

---

## НАЗНАЧЕНИЕ

Удаление только доказанно неиспользуемого кода.

Не удалять по имени.  
Не удалять "на всякий случай".  
Только после проверки.

---

## ПРАВИЛА

### 1. Доказательства перед удалением
Перед удалением файла/кода обязательно:
- [ ] Найти все импорты этого файла
- [ ] Найти все вызовы функций/классов
- [ ] Найти все ссылки в HTML/CSS/JS
- [ ] Найти все упоминания в документации
- [ ] Найти все упоминания в конфигах (nginx, docker, package.json)
- [ ] Confidence score 100% (или объяснение почему меньше)

### 2. Процесс удаления

#### Шаг 1: Поиск зависимостей
```bash
# Поиск импортов
grep -r "from .* import" . | grep <имя_файла>
grep -r "require.*<имя_файла>" .
grep -r "import.*<имя_файла>" .

# Поиск вызовов функций
grep -r "<имя_функции>" . --include="*.py" --include="*.js" --include="*.jsx"

# Поиск в конфигах
grep -r "<имя_файла>" docker-compose.yml nginx.conf package.json requirements.txt

# Поиск в документации
grep -r "<имя_файла>" docs/
```

#### Шаг 2: Анализ результатов
| Результат | Действие |
|-----------|----------|
| 0 зависимостей | Можно удалять (Confidence 100%) |
| 1-2 зависимости (в том же файле) | Можно удалять (Confidence 90%+) |
| Зависимости в других модулях | НЕ удалять, перенести в archive/remove-candidate/ |
| Зависимости в production коде | НЕ удалять |

#### Шаг 3: Безопасное удаление
```bash
# 1. Переместить в временную папку
mkdir -p archive/remove-candidate
git mv <файл> archive/remove-candidate/

# 2. Запустить проект
# Backend
cd app/backend && uvicorn main:app --reload

# Frontend
cd nestandart-phuket && npx serve . -p 3000

# 3. Проверить работоспособность
curl http://localhost:8000/api/v1/health
curl http://localhost:3000

# 4. Если всё работает — удалить
rm archive/remove-candidate/<файл>

# 5. Если что-то сломалось — вернуть
git mv archive/remove-candidate/<файл> <оригинальный_путь>
```

### 3. Удаление кода (не файлов)

#### Функции/классы
1. Найти все вызовы: `grep -r "<имя_функции>" .`
2. Если 0 вызовов — можно удалять
3. Если есть вызовы — НЕ удалять

#### CSS классы
1. Найти все использования: `grep -r "\.<имя_класса>" .`
2. Проверить в DevTools Coverage (Chrome)
3. Если не используется — можно удалять

#### API endpoints
1. Найти все вызовы: `grep -r "/api/v1/<endpoint>" .`
2. Проверить в n8n workflows
3. Проверить в frontend (fetch/axios)
4. Если не используется — можно удалять

## ЧТО ЗАПРЕЩЕНО

| Запрещено | Почему |
|-----------|--------|
| Удалять по имени | Нет доказательств |
| Удалять "на всякий случай" | Может сломать |
| Удалять без проверки импортов | Риск сломать |
| Удалять без запуска проекта | Неизвестно, работает ли |
| Массовое удаление | Невозможно проверить |
| Удалять из `archive/remove-candidate/` сразу | Нужно протестировать |

## КОНТРОЛЬНЫЕ ВОПРОСЫ

Перед удалением:
1. **Есть доказательства?** (все импорты/вызовы проверены)
2. **Никто не использует?** (0 зависимостей)
3. **Проект запускается?** (после удаления)
4. **Основной функционал работает?** (сайт, бот, API)
5. **Можно откатить?** (`git revert` работает)

Если хотя бы один ответ «нет» — **НЕ УДАЛЯТЬ**.
